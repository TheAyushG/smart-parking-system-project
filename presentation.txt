Slide 2: Problem Statement
- Traffic Congestion: Drivers spend huge amounts of time circling for parking, causing stress and burning fuel.
- Unreliable Availability: Traditional parking lots have no real-time availability tracking.
- Overstay Abuse: Drivers often park longer than they paid for, blocking spots for other users.
- Booking Clashes: Double-booking errors occur when digital systems aren't synchronized with physical slots.

Slide 3: Proposed Solution
- Real-Time Booking Platform: A centralized web app where users can find and pre-book guaranteed parking spots before they arrive.
- Interactive Maps: Visual map interfaces showing live availability of parking locations across the city.
- Automated Enforcement: Dynamic penalty timers that automatically charge users if they overstay their booked time.
- Instant Synchronization: When someone books a slot, the spot is instantly marked as "Occupied" on every other user's screen worldwide.

Slide 4: Core Features & Functionalities
* Authentication: Secure user Login/Registration and Password Recovery.
* Smart Search: Map-based dynamic location searching.
* Real-Time Slot Engine: Live 100% accurate grids showing exactly which slot numbers are available or booked.
* Pre-Booking: Users can select Future Start Times to reserve slots in advance.
* Live Parking Timers: The "My Bookings" dashboard provides a live countdown timer when a user is parked.
* Dynamic Overstay Penalties: If the timer hits zero, a red count-up timer begins dynamically recalculating their penalty price (1.5x standard rate per hour).
* Automated Payments: Integrated digital checkout flow.

Slide 5: Technologies Used (The Tech Stack)
Frontend (Client Side):
- React.js: Core UI framework.
- React-Leaflet / OpenStreetMap: Interactive mapping system to render GPS pins.
- Socket.io-client: Enables live multiplayer-style updates to the UI without page reloads.

Backend (Server Side):
- Node.js & Express.js: Fast, scalable backend REST API framework.
- Socket.io (Server): Broadcasts WebSockets to instantly update slot availability everywhere.
- Node-Cron: Background job scheduler.

Database & Security:
- MongoDB & Mongoose: NoSQL database for flexible Slot, User, and Booking document storage.
- JWT (JSON Web Tokens): Secure route authentication.
- Bcrypt.js: Password encryption.

Slide 6: Key API Integrations
1. Razorpay Payment Gateway API:
   - Replaces fake card forms with a secure, real-world native payment popup.
   - Uses cryptographic HMAC signatures (crypto.createHmac) on the Backend to securely verify the payment success token before granting the booking.
2. Google Places Photo API (Architecture):
   - Automatically attempts to fetch real-world location backdrop photographs based on your search query.
   - Includes a robust fallback engine that injects beautiful placeholders (picsum.photos) if API limits are reached.
3. Nominatim OpenStreetMap API:
   - Powers the search bar to convert text locations (e.g., "Taj Mahal") into exact GPS map coordinates.

Slide 7: The True "Clean Slate" Architecture
- Volatile RAM Maps: When a user searches for a custom location, it is stored purely in the React memory hierarchy (Volatile RAM) rather than the database. When the browser refreshes, it vanishes keeping the core database clean.
- System Wipe Rebooting: The backend initialization script is built to automatically recreate exactly 6 Permanent Locations (with 100% empty slots) guaranteeing the map is pristine when the server deploys.

Slide 8: Real-Time Flow Demonstration (Live Notifications)
